# Active 资产只保留最终版

**问题描述**：用户明确要求整理项目资产：所有内容包括提示词、图片等，只保留最终版本；旧的、过程中的、候选和临时产物都要归档。后续做新内容也必须遵守同一规则。

**触发场景**：动漫短剧生产中会不断生成候选角色图、关键图 v01/v02、图片生成提示词、可灵提示词、旧审查记录和临时素材。如果这些过程产物留在 active 目录，会让后续生产引用错误版本，增加混乱和返工。

**教训/建议**：后续生成任何动漫资产时，active 目录只保留最终使用版本，文件名用稳定名且不带 `v01/v02/candidate/draft/old`。过程产物一律移动到 `98-archive/` 并尽量保留原相对路径；只有用户确认最终版后才进入 active。生成关键图、可灵提示词、字幕、音频脚本、角色图、视频片段都适用这个规则。
