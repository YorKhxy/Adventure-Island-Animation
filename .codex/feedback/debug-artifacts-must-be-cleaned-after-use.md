# 调试和测试临时产物用完必须清理

## Context

用户发现项目根目录出现 `.playwright-mcp/`，这是浏览器/Playwright MCP 调试可灵页面时自动生成的控制台日志和页面快照。用户要求删除，并明确以后调试测试产生的东西，调试完测试完要删除。

## Lesson

调试产物不是项目资产。浏览器快照、控制台日志、截图、临时导出、review frame、cache 目录等如果不主动清理，会污染工作区，让用户分不清哪些文件该提交、哪些只是工具垃圾。

## Rule

- 使用 Playwright、浏览器 MCP、ffmpeg 截帧、日志导出、临时截图或任何调试工具后，结束前必须检查是否产生临时文件/目录。
- 临时调试产物如果不是明确交付物，必须删除。
- 如果临时产物需要保留作为证据，必须放到明确的审查/归档路径并说明用途，不能丢在项目根目录。
- 提交前必须用 `git status --short` 确认没有调试垃圾混在未跟踪文件里。
