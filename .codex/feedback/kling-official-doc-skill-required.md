# 可灵提示词必须走官方文档 Skill

**问题描述**：用户提供了可灵官方文档，并明确要求根据文档提炼成 Skill，以后每次需要生成可灵提示词时都使用该 Skill。此前流程中可灵提示词容易被当成通用 AI 视频提示词处理，缺少对可灵视频3.0能力的专门利用。

**触发场景**：动漫短剧生产中，需要把脚本、分镜、关键图、首帧/尾帧、角色参考图或配音台词转成可灵AI提示词时。如果不先加载专用 Skill，就容易忽略可灵3.0的智能分镜、主体参考、三人及以上指代、音画同步、15秒生成和多语种能力。

**教训/建议**：后续凡是出现“可灵AI / 可灵 / Kling / 可灵视频3.0 / 图生视频 / 主体参考 / 音画同步提示词”等需求，必须先 read `.codex/skills/kling-prompt-builder/SKILL.md`，再输出提示词。若还没有动态分镜，先拆镜头；若有关键图和角色图，优先生成“图生视频 + 主体参考”格式；默认使用中文提示词。
