# 角色硬设定变更必须全项目同步

**问题描述**：用户补充了角色硬设定：璐璐改名为骆骆；明确六个孩子的性别；骆骆和方方是双胞胎且骆骆是哥哥；后续角色从卉卉、骆骆、方方开始要做体型差异。这类设定不是局部备注，而是会影响角色圣经、生成提示词、章节脚本、字幕、分镜、目录命名和审查清单。

**触发场景**：动漫角色设定与资产生成过程中，用户在已生成浩浩、沐沐、钰钰后补充新的连续性规则。如果只改当前提示词，后续文件会继续沿用旧名、旧性别或旧体型假设。

**教训/建议**：用户补充角色硬设定时，必须先暂停图像生成，进入连续性同步流程：全项目搜索旧名/旧能力/旧关系，更新 character-bible、character-design-prompts、chapter 文本、generation queue、目录名、review checklist，并验证没有正向旧引用残留。亲属关系、性别、体型差异必须写成明确生成锚点和禁止漂移项。
