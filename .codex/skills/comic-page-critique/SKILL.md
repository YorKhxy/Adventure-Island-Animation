[任务]
    审查漫画页或条漫的阅读体验和画面表现，发现分格、节奏、角色表演、对白和视觉层级问题。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 用户提供漫画页、条漫截图、分镜表或漫画脚本。

    可选：
    - 阅读方向、目标平台、画风参考、脚本原文。

    降级策略：
    - 没有成图时审查分镜文字和对白结构。
    - 没有阅读方向时，默认按中文移动端竖向阅读审查。

[第一性原则]
    **阅读动线第一**：读者不知道先看哪里，画得再漂亮也废。
    **分格服务节奏**：大格小格、远景近景、留白都要服务情绪和信息量。
    **角色要会演**：表情、姿态、视线和距离要表达关系，不是摆人偶。
    **对白不能堵画面**：气泡位置、字数和画面重心必须协调。
    **联网优先**：平台条漫规格、阅读习惯、漫画排版规范不确定时先搜索确认。

[文件结构]
    ```
    comic-page-critique/
    └── SKILL.md
    ```

[漫画审查维度]
    必须检查：
    - 阅读顺序是否明确
    - 每格信息是否过载
    - 分格大小是否体现重点
    - 镜头景别是否有变化
    - 角色表演是否清楚
    - 对白气泡是否挡关键画面
    - 字幕/对白是否太长
    - 黑白灰或色彩层级是否清楚
    - 动作连续性是否断裂
    - 情绪转折是否有停顿

[工作流程]
    [第一步：确认阅读形态]
        判断是页漫、条漫、四格、分镜草稿还是动漫短剧分镜。

    [第二步：检查阅读动线]
        从第一眼焦点到最后一格，检查读者视线是否顺。

    [第三步：检查分格节奏]
        判断关键情绪和动作有没有被放大，过渡有没有缺失。

    [第四步：检查角色表演]
        审查表情、姿态、距离、视线和动作连续性。

    [第五步：输出修改建议]
        给出删格、并格、放大、重排、改对白或重画建议。

[输出格式]
    - **审查结论**：PASS / PASS_WITH_FIXES / FAIL
    - **阅读形态判断**
    - **阅读动线问题**
    - **分格节奏问题**
    - **角色表演问题**
    - **对白/气泡问题**
    - **重排建议**
    - **复审重点**

[初始化]
    执行 [第一步：确认阅读形态]。
