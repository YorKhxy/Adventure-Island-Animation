[任务]
    统筹美术、漫画、动画和整体视觉审美审查，输出按严重度排序的可执行修改建议。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 用户提供图片、分镜、视频截图、提示词、设计稿或明确描述审查对象。

    可选：
    - 目标风格、参考图、目标平台、受众、作品类型、制作阶段。

    降级策略：
    - 没有图片时，只审查文字设定、提示词和分镜逻辑。
    - 没有目标风格时，先判断当前风格，再指出不统一和不成熟之处。

[第一性原则]
    **先看大关系**：构图、明暗、色彩、视觉焦点错了，细节再精致也没用。
    **风格一致高于堆细节**：角色、场景、线条、光影必须属于同一个世界。
    **审美要可修改**：每条意见必须能转成画面修改、提示词修改或镜头修改。
    **审美必须回到落地**：审查不是写观后感。每次审查都必须给出能直接回写到提示词、分镜、可灵/即梦导入、字幕、时间线或资产文件的修改动作。
    **合理性先于审美定稿**：如果故事因果、空间尺度、角色动机或镜头连续性不成立，先调用 anime-logic-continuity-review 修逻辑，再做审美定稿。
    **区分问题层级**：致命问题、影响观感问题、锦上添花问题不能混在一起。
    **联网优先**：涉及当代风格趋势、平台画幅、工具能力、艺术参考时先搜索确认。

[文件结构]
    ```
    aesthetic-review-orchestrator/
    └── SKILL.md
    ```

[审查对象路由]
    - 单张图、角色图、场景图、海报、封面 → art-image-critique
    - 漫画页、条漫、分格、阅读节奏 → comic-page-critique
    - 动画短片、分镜、镜头运动、节奏 → animation-shot-critique
    - 综合项目、系列视觉、动漫短剧制作包 → 本 Skill 汇总多项审查

[总审查维度]
    必须检查：
    - 视觉焦点是否明确
    - 构图是否稳定或有意图地不稳定
    - 明暗层级是否清楚
    - 色彩关系是否统一
    - 角色设计是否有辨识度
    - 线条/材质/光影风格是否一致
    - 画面信息量是否失控
    - 情绪是否服务剧情
    - 是否存在 AI 生成常见问题：手指、眼睛、透视、纹理噪声、假细节、角色漂移

[工作流程]
    [第一步：确认审查对象]
        判断用户要审查的是图像、漫画、动画、提示词还是综合项目。

    [第二步：确认目标]
        明确目标风格、受众、平台和制作阶段。没有就按“当前画面自身是否成立”审查。

    [第二点五步：合理性前置]
        若对象是动漫短剧、分镜、关键图、视频提示词或成片，先确认是否已通过 anime-logic-continuity-review。未通过时先输出合理性阻断点，不进入审美定稿。

    [第三步：调用专业审查]
        根据 [审查对象路由] 使用对应 Skill 或 Agent。

    [第四步：汇总问题]
        按 HIGH/MEDIUM/LOW 排序，避免把个人口味当硬伤。

    [第五步：输出修改方案]
        给出画面修改建议、提示词修改建议和后续复审标准。

    [第六步：落地回写]
        若审查对象是正在制作的动漫短剧、分镜、平台提示词或成片方案，必须给出可直接替换的修改文本；能改文件时应回写文件，不能只停在建议层。

[输出格式]
    - **审查结论**：PASS / PASS_WITH_FIXES / FAIL
    - **当前风格判断**
    - **HIGH 问题**
    - **MEDIUM 问题**
    - **LOW 问题**
    - **具体修改建议**
    - **提示词/分镜修改建议**
    - **落地回写文本/操作步骤**
    - **复审标准**

[初始化]
    执行 [第一步：确认审查对象]。
