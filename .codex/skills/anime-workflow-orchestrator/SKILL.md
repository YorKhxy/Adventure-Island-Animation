[任务]
    编排“小说文本 -> 动漫短剧制作稿 -> 素材生成指令 -> mp4 导出准备”的完整半自动流程。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 用户提供小说文本、章节文本或明确的公版作品片段。
    - 用户确认本次制作风格、目标平台和是否需要导出 mp4。

    可选：
    - 角色参考图、画风参考图、已有 ComfyUI 工作流、TTS 偏好、视频比例。

    降级策略：
    - 未提供参考图时，用文本角色设定和固定提示词维持一致性。
    - 未提供本地工具信息时，只输出制作包，不假装已经能渲染视频。

[第一性原则]
    **版权先行**：只处理用户有版权、已授权或公版文本，不协助洗稿搬运。
    **半自动优先**：每个关键节点都保留人工确认，别让 AI 胡编一路跑到底。
    **动漫而非漫画**：输出必须覆盖动作、镜头、转场、配音、字幕和视频节奏，不只写静态分镜。
    **可制作优先**：每个产物都要能交给 ComfyUI、TTS 或 ffmpeg 使用。
    **联网优先**：涉及工具版本、模型授权、平台规格时先 WebSearch 确认再建议。

[文件结构]
    ```
    anime-workflow-orchestrator/
    └── SKILL.md
    ```

[流程阶段]
    必须按以下顺序推进，不要跳步：
    1. 文本摄取：调用 anime-source-ingest，确认来源、版权、章节范围和切片策略。
    2. 剧本改编：调用 anime-script-adapter，生成短剧结构、解说稿和剧情节奏。
    3. 连续性设定：调用 anime-continuity-bible，固定角色、世界观、口癖、视觉锚点。
    4. 动漫分镜：调用 anime-shot-designer，拆镜头、动作、运镜、时长和转场。
    5. 视觉提示词：调用 anime-visual-prompt-builder，输出图片/视频生成提示词和一致性策略。
    6. 配音字幕：调用 anime-voice-subtitle-director，输出旁白、角色语气、TTS 分段和字幕。
    7. 合成导出：调用 anime-assembly-exporter，输出素材清单、时间线和 ffmpeg 合成计划。
    8. 成片审查：调用 anime-production-review，检查剧情、节奏、版权、视觉一致性和导出风险。

[人工确认节点]
    以下节点必须停下来让用户确认或修改：
    - 剧情提炼确认
    - 短剧脚本确认
    - 分镜/画面提示词确认
    - 配音风格确认
    - 最终视频预览确认

[输出格式]
    默认输出为“动漫短剧制作包”，包含：
    - 项目标题
    - 源文本摘要
    - 改编策略
    - 角色和场景设定
    - 解说稿
    - 分镜表
    - 视觉提示词表
    - 配音和字幕表
    - 素材清单
    - 合成导出计划
    - 审查问题清单

[初始化]
    执行 [依赖检测]，然后按 [流程阶段] 推进。
