[任务]
    对动漫短剧制作包或成片预览做审查，发现剧情、节奏、视觉、声音、版权和制作可行性问题，并给出修复建议。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 至少提供脚本、分镜、提示词、配音字幕或成片预览中的一种。

    可选：
    - 原文片段、连续性圣经、生成素材、导出视频。

    降级策略：
    - 缺少原文时，只审查制作质量，不判断忠实度。
    - 缺少成片时，只审查制作包，不判断最终视听效果。

[第一性原则]
    **先找问题**：审查不是夸稿子，先指出会毁片的问题。
    **按严重度排序**：版权、剧情错误、角色崩坏、生成不可行优先。
    **证据明确**：每个问题要引用对应镜头、段落或提示词。
    **给可执行修复**：不要只说“不够好”，要说改哪里、怎么改。
    **制作合理性审美落地闭环**：动漫制作包或成片预览审查后，必须衔接合理性审查、审美审查和落地修复。发现故事因果、空间尺度、角色动机、镜头衔接、视觉、动作、转场、AI味或平台导入风险时，要明确调用 anime-logic-continuity-review、aesthetic-review-orchestrator 或 animation-shot-critique，并把修复写回提示词、分镜或导入步骤。
    **联网优先**：版权、公版、平台规则、工具能力不确定时先搜索确认。

[文件结构]
    ```
    anime-production-review/
    └── SKILL.md
    ```

[审查维度清单]
    Stage 1 内容合规：
    - 来源和版权边界
    - 是否忠于原文核心
    - 是否乱编关键事实
    - 是否有平台敏感风险

    Stage 2 制作质量：
    - 钩子是否足够强
    - 节奏是否拖沓
    - 镜头是否可制作
    - 角色是否一致
    - 提示词是否可生成
    - 配音和字幕是否自然
    - 时间线是否可合成

[工作流程]
    [第一步：确认审查对象]
        判断当前审查的是脚本、分镜、视觉提示词、配音字幕、合成计划还是成片。

    [第二步：Stage 1 内容合规]
        先审查版权、忠实度、平台敏感和核心剧情错误。严重问题存在时先停，不进入制作质量粉饰。

    [第三步：Stage 2 制作质量]
        审查节奏、动漫感、镜头可制作性、角色一致性、提示词质量和音画同步。

    [第四步：输出修复清单]
        按 HIGH/MEDIUM/LOW 排序，给出具体修改建议。

    [第五步：合理性、审美与落地闭环]
        若审查对象包含剧情、关键图、视频提示词、动态分镜、成片或平台导入方案，必须补充合理性复查结论、审美复查结论和落地修复文本。不能只输出“建议优化”。

[输出格式]
    - **审查结论**：PASS / PASS_WITH_FIXES / FAIL
    - **HIGH 问题**
    - **MEDIUM 问题**
    - **LOW 问题**
    - **修复建议**
    - **合理性补救动作**
    - **审美/落地补救动作**
    - **复审要求**

[初始化]
    执行 [第一步：确认审查对象]。
