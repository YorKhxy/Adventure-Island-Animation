---
name: anime-script-adapter
description: 当需要把小说片段改编成动漫短剧脚本、解说稿、剧情钩子和短视频节奏时使用。产出可进入分镜制作的短剧剧本。
---

[任务]
    将小说素材改编成 45-120 秒动漫短剧脚本，包含钩子、剧情推进、解说、人物台词和结尾悬念。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - anime-source-ingest 产出的改编输入摘要，或用户提供的小说文本片段。

    可选：
    - 用户指定画风、目标平台、时长偏好、叙事口吻。

    降级策略：
    - 未指定时长时，由剧情密度自动决定，默认 45-120 秒。
    - 未指定风格时，先给 3 个风格方向让用户选，不替用户装懂。

[第一性原则]
    **短视频先钩住**：前 3 秒必须有冲突、悬念或强画面。
    **改编不是复述**：保留原作核心，但用短剧节奏重组信息。
    **解说服务画面**：旁白不能把画面说死，要给动作和镜头留空间。
    **动漫化表达**：增强动作、表情、环境、视觉冲击和转场节奏。
    **联网优先**：涉及平台节奏、热点叙事模板、题材限制时先搜索确认。

[文件结构]
    ```
    anime-script-adapter/
    └── SKILL.md
    ```

[剧本维度清单]
    必须包含：
    - 标题
    - 一句话卖点
    - 前 3 秒钩子
    - 剧情三段结构：开场压迫、冲突升级、反转/悬念
    - 解说旁白
    - 角色台词
    - 动漫化动作描述
    - 推荐时长
    - 结尾引导

    禁止：
    - 只做小说摘要
    - 大段心理描写无画面
    - 角色动机被改到违背原文
    - 为了爽点乱编核心事实

[工作流程]
    [第一步：确定改编策略]
        判断本段适合悬疑、热血、搞笑、史诗、悲情还是反转风格。

    [第二步：设计钩子]
        生成 2-3 个前 3 秒开场方案，并推荐一个最适合短视频的平台型钩子。

    [第三步：写短剧脚本]
        输出时间段、画面意图、解说、台词和情绪推进。

    [第四步：压缩和校正]
        删除无画面价值的信息，保留可视化冲突和人物选择。

    [第五步：交付分镜输入]
        把剧本整理成 anime-shot-designer 可以直接拆镜的结构。

[输出格式]
    - **短剧标题**
    - **推荐时长**
    - **一句话卖点**
    - **前 3 秒钩子**
    - **分段剧本表**：时间段、剧情、解说、台词、情绪
    - **结尾悬念/引导**
    - **给分镜的制作备注**

[初始化]
    执行 [第一步：确定改编策略]。
