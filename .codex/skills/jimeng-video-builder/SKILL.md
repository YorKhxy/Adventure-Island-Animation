[任务]
    将动漫短剧片段、分镜、关键图、角色设定、台词和字幕需求转成可直接在即梦AI视频生成界面执行的操作清单与中文提示词。输出必须按即梦界面填写顺序组织，所有提示词直接给可复制文本。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 用户明确提到即梦、Jimeng、即梦AI、去即梦导入、用即梦生成视频，或上下文目标平台已切换为即梦。
    - 至少有一个视频片段目标：脚本段落、分镜、关键图说明、首帧图、尾帧图或用户口述画面。

    可选：
    - 首帧关键图、尾帧关键图、角色参考图、场景参考图。
    - anime-shot-designer 产出的动态分镜表。
    - anime-continuity-bible 产出的角色固定设定。
    - 配音台词、旁白、字幕时轴、SRT/ASS 文件。
    - 目标比例、时长、是否要首尾帧控制、是否要生成字幕。

    降级策略：
    - 无首帧图时，输出文生视频模式，并明确建议补关键图以提高角色一致性。
    - 只有首帧图无尾帧时，输出图生视频模式，不假装有首尾帧控制。
    - 有首帧和尾帧时，优先输出首尾帧视频模式，提示首帧控制开场构图、尾帧控制落点。
    - 无台词时，不伪造对白；只输出画面提示词和后期字幕建议。
    - 即梦界面无独立负面提示词框时，将负面约束合并到主提示词末尾的“禁止事项”。

[第一性原则]
    **按界面交付**：用户站在即梦界面前，需要的是上传什么、点什么、填什么；不要只给文件路径、行号或文档地址。
    **中文直接可用**：即梦支持中文创作，默认用中文提示词，句子要明确、结果导向、可直接粘贴。
    **首尾帧优先控结果**：即梦支持文/图生视频和首帧到尾帧控制；有首尾帧时不要用纯文生硬赌画面。
    **一段只讲一件事**：单次生成只承载一个核心剧情动作，复杂段落拆成多个节拍或多个短片段。
    **字幕后期优先**：除非用户明确要即梦内生字幕，否则动漫短剧默认不让画面生成字幕，避免伪字、错字和遮挡。
    **联网优先**：涉及即梦平台能力变化、入口变化、模型参数变化时先联网确认再输出。

[即梦能力摘要]
    当前按已联网确认的即梦AI公开能力处理：
    - 即梦AI支持文生视频和图生视频。
    - 即梦AI支持首帧图片和尾帧图片输入，用于控制视频从开场到结尾的变化。
    - 即梦AI支持中文提示词创作。
    - 即梦AI强调流畅运镜、生动自然的动效，可在提示词中写镜头运动和速度变化。
    - 即梦AI与剪映生态相连，字幕、配音、剪辑、去水印终版处理更稳时可交给后期完成。

[即梦输入维度清单]
    每次交付必须判断并输出：
    - 生成模式：文生视频 / 图生视频 / 首尾帧视频。
    - 基础设置：比例、时长、清晰度或模型选项，如用户未指定则给推荐值并标明可改。
    - 上传素材：首帧图、尾帧图、角色参考图、场景参考图、音频或字幕文件。
    - 素材用途映射：每张图具体负责什么，不能含混写“都参考”。
    - 主提示词：画风、角色、场景、动作、镜头、光影、节奏、结尾落点。
    - 负面约束：水印、伪文字、角色漂移、动作同步、恐怖化、乱切、过曝等。
    - 台词/字幕策略：即梦内生字幕或后期压字幕；默认后期压字幕更稳。
    - 导出命名：建议保存文件名和 active/归档规则。
    - 通过标准：生成后要看哪些点，失败就重做哪些点。

[即梦提示词策略]
    **模式选择**：
    - 只给文字想法 → 文生视频。
    - 有开场关键图 → 图生视频，提示词必须写“以首帧图为第一帧和构图锚点”。
    - 有开场和结尾关键图 → 首尾帧视频，提示词必须写“从首帧自然过渡到尾帧”。
    - 角色一致性要求高但即梦无主体绑定时 → 用首帧锁构图，提示词中用角色姓名、位置、服装色彩、动作差异强化，不承诺主体绑定。

    **素材映射写法**：
    - 首帧图负责开场构图、角色初始位置、场景和画风。
    - 尾帧图负责结尾构图、角色最终状态和情绪落点。
    - 角色参考图只负责身份、发型、服装和色彩，不负责镜头运动。
    - 场景参考图只负责环境风格，不改变角色身份。

    **提示词结构**：
    - 第一段写生成目标和剧情节奏。
    - 第二段写画风、光影、角色一致性。
    - 第三段按时间轴写动作和运镜。
    - 第四段写结尾落点和衔接下一段。
    - 最后一段写禁止事项；如果即梦有负面提示词框，则把禁止事项单独放入负面框。

    **时间轴写法**：
    - 5秒以内写2个节拍。
    - 6-10秒写2-3个节拍。
    - 10-15秒写3-4个节拍。
    - 超过15秒建议拆段，不强塞。

    **字幕和声音策略**：
    - 即梦视频提示词默认不生成字幕，字幕用剪映/SRT后期压制。
    - 用户明确要内生字幕时，必须写“只显示指定字幕，不生成额外文字”，并提醒错字风险。
    - 台词用于指导表演、口型和剪辑节奏时，可以放在主提示词末尾，但不要让无关角色开口。

[输出格式]
    每次用户问“怎么导入即梦 / 即梦怎么生成 / 用即梦生成视频”时，必须按以下顺序输出：

    1. **即梦基础设置**：模式、比例、时长、是否首尾帧、字幕策略。
    2. **上传素材顺序**：逐条列出本地路径和上传位置，不贴文档地址。
    3. **素材用途映射**：首帧、尾帧、角色参考、场景参考分别怎么用。
    4. **主提示词**：一个可直接复制的 `text` 代码块。
    5. **负面提示词**：如果界面有负面框，给独立 `text` 代码块；如果没有，说明已合并到主提示词。
    6. **台词/字幕/后期**：给可复制台词，说明是否在即梦生成字幕或后期压字幕。
    7. **生成后检查**：列出通过标准和失败重做点。
    8. **保存命名**：给建议输出路径和文件名。

    禁止输出：
    - 不要用“参考某某文档链接”代替具体操作。
    - 不要只给提示词文件路径或行号。
    - 不要让用户自己从长文档拆主提示词、负面提示词和分镜。
    - 不要把可灵的主体绑定、音画同步能力照搬到即梦，除非即梦界面当前明确提供同等控件。

[工作流程]
    [第一步：识别即梦模式]
        判断是文生视频、图生视频还是首尾帧视频。优先使用用户已有关键图；信息不足时给最稳降级方案。

    [第二步：整理素材]
        从项目上下文中找首帧、尾帧、角色参考、字幕和音频。输出给用户时必须按上传顺序列明本地路径和用途。

    [第三步：重写即梦提示词]
        把脚本、分镜和角色设定转成即梦适配提示词：中文、时间轴明确、运镜清楚、负面约束可复制。

    [第四步：按界面顺序交付]
        按 [输出格式] 输出，所有需要粘贴的内容直接放入代码块，不用文档地址代替。

    [第五步：生成后审片]
        用户提供即梦生成视频后，检查角色一致性、镜头节奏、字幕文字、水印、画面文字、结尾落点和是否可进入最终合成。

    执行 [第一步：识别即梦模式]。
